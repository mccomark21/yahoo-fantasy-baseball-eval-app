## Anti-Patterns Verdict

LLM assessment: Loading copy leaks "DuckDB" implementation detail. Dark mode toggle is canonical shadcn starter-kit output. border-b header with no background signals no design decision was made. Error state is a one-liner placeholder. Template tells present.

Deterministic scan: Returned [] — no automated slop patterns found. Detector operates on CSS/markup patterns; JSX class strings not matched. Shell design issues are structural and color-system violations, not template-pattern violations.

## Overall Impression

Structurally sound shell (h-dvh, flex min-h-0, overflow-hidden are correct) but reads as unfinished kit scaffold. The one intentional design decision (nav strip) uses the explicitly retired teal palette. Everything else is shadcn zero-effort defaults. Single focused pass would fix the most critical issues.

## What's Working

1. Source metadata bar — surfacing data provenance (title, date, source link) inside the UI is analytically honest and genuinely useful.
2. Shell layout foundation — h-dvh, flex min-h-0, overflow-hidden correctly handles iOS Safari and prevents table overflow.
3. Responsive header padding — px-3 py-2 md:px-4 md:py-3 shows mobile care.

## Priority Issues

[P0] Teal nav strip — bg-teal-950, text-teal-100, aria-pressed:bg-teal-700 at App.tsx:427–459. Directly violates DESIGN.md's most explicit prohibition. Fix: Replace with bg-[#002D62], active tab uses 2px red bottom border (#BF0D3E), text-white/70 default. Command: /impeccable craft header.

[P0] Error state dead end — App.tsx:382–391. Raw {error} string, text-destructive, no retry. Fix: Plain-language message + Retry button + collapsed technical detail. Command: /impeccable harden.

[P1] Header has no background — header has border-b only, no background color or identity. DESIGN.md specifies MLB Navy header band as the identity anchor. Fix: Apply surface tint or full navy band. Command: /impeccable craft header.

[P1] Loading state leaks DuckDB + no skeleton — "Initializing DuckDB and fetching data..." is implementation copy. No skeleton for 3–8s load. Fix: Remove DuckDB from copy; add skeleton layout. Command: /impeccable harden.

[P2] ToggleGroup wrong semantics — aria-pressed on mutually exclusive nav items breaks WCAG 4.1.2. Fix: Replace with role=tablist/tab or proper nav+button pattern. Command: /impeccable audit.

## Persona Red Flags

Alex (Power User): No keyboard shortcuts for tab switching. "Injured" tab ambiguous. Dark mode toggle occupying prime header real estate permanently.

Sam (Accessibility): ToggleGroup aria-pressed announced wrong by screen readers. Loading state has no role=status/aria-live. Error state has no role=alert. Focus not moved to new content on tab switch.

Riley (Stress Tester): Indefinite loading state with no timeout signal. Per-view errors shown in text-xs muted — easy to miss. Rapid tab switching could stress query scheduling race conditions.
